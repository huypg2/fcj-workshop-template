# Summary Report “FCJ Community Day - From Zero to Cloud Hero”

## 1. Event overview

**FCJ Community Day - From Zero to Cloud Hero** was an event that shared many practical topics related to application development, system operations, artificial intelligence, and cybersecurity on the AWS platform. The topics in the program helped participants gain a broader view of how to build modern systems, from real-time communication, application containerization, context processing for LLMs, cyber attack detection, to career development in infrastructure and DevOps.

The event did not focus only on theory, but also connected the content with practical deployment models, commonly used tools, and real working experience in the technology field.

## 2. Program content

During the event, the speakers shared many different topics.

**Nguyễn Quốc Bảo** presented the topic **“Multiplayer in the Cloud: Connecting Godot Clients with AWS WebSockets”**. The sharing session explained how game clients can communicate in real time in a cloud environment. The content emphasized the role of WebSocket and AWS API Gateway in maintaining two-way connections, which is suitable for multiplayer applications that require fast responses.

**Bảo Huỳnh** presented the topic **“Docker – A containerization technology”**. The content focused on explaining how Docker packages an application together with its runtime environment, making deployment more consistent. The presentation also compared containers with virtual machines, highlighting the advantages of faster startup time, better resource efficiency, and flexibility in DevOps workflows.

**Việt Phát** presented the topic **“GraphRAG: Build GraphRAG applications using Amazon Bedrock and Amazon Neptune”**. The presentation introduced an approach to building AI applications by combining Amazon Bedrock with Amazon Neptune. The content showed how Knowledge Graph and Vector Database can help LLMs retrieve context more effectively, thereby improving the accuracy of responses.

**Lê Hoàng Gia Đại** presented the topic **“WAF + ML for Cyber Attack Detection: Machine Learning-based Network Intrusion Detection System (NIDS) on AWS”**. The content focused on applying Machine Learning to cyber attack detection on AWS. The speaker explained how the LightGBM model can be combined with AWS WAF to identify abnormal behaviors and strengthen system protection.

**Trần Trung Vinh** presented the topic **“From IT Helpdesk to Senior Sysadmin”**. The sharing session focused on the career development journey in system administration. The content helped participants better understand the transition from IT Helpdesk to Senior Sysadmin, as well as the skills needed to operate On-Premise infrastructure and approach modern system operation thinking.

## 3. Notable points

The first notable point of the event was the sharing session about WebSocket on AWS. This content showed that real-time applications such as multiplayer games, chat systems, or status update systems can be built in a serverless direction, reducing the need to manage servers and improving scalability.

The Docker topic was also very practical because containers are an important technology in modern application deployment. Understanding images, containers, layers, volumes, and networks helps the process of packaging, testing, and deploying applications become more stable across different environments.

Another notable point was GraphRAG. This topic expanded my understanding of GenAI, especially how to combine LLMs with relational data. Using Knowledge Graph helps AI systems better understand the relationships between entities instead of relying only on similar text passages.

In addition, the topic about security with AWS WAF and Machine Learning showed the trend of combining multiple defense layers in modern systems. Instead of only blocking requests based on fixed rules, the system can also analyze data to detect abnormal behaviors.

The sharing session about the journey from IT Helpdesk to Senior Sysadmin helped participants understand that developing in infrastructure and DevOps requires building a strong foundation step by step, starting from troubleshooting skills, system administration, computer networking, automation, and cloud.

## 4. Lessons learned

After the event, I understood more clearly the role of AWS in modern systems. AWS can support not only common web application deployment, but also real-time games, AI applications, security systems, and large-scale operational infrastructure.

I also learned that Docker is an important tool when working on software projects. Packaging applications with containers helps reduce environment-related errors, supports faster deployment, and fits well with DevOps models.

Besides that, I learned more about GraphRAG and how Knowledge Graph can support LLMs in processing information more effectively. This is a useful approach when building AI applications that require high accuracy and rich context.

In terms of security, I realized that a system needs multiple protection layers. Combining AWS WAF with Machine Learning can help detect both known attack patterns and abnormal behaviors that have not been clearly defined by fixed rules.

For career orientation, the event helped me understand that fundamental skills are very important. To go further in Cloud or DevOps, it is necessary to practice Linux, Networking, Docker, CI/CD, Infrastructure as Code, monitoring, and troubleshooting skills.

## 5. Application after the event

After the event, I can apply Docker knowledge to package services in my project, helping the application run more consistently when moving from the local environment to the deployment environment.

I can also learn more about WebSocket on AWS to apply it to features that need real-time data updates, such as system status dashboards, notifications, or process tracking.

With the knowledge of GraphRAG, I have a better direction when learning about GenAI applications that use private data. Combining Amazon Bedrock, Amazon Neptune, and Knowledge Graph is a suitable approach for systems that need to retrieve information with complex relationships.

In terms of security, I can continue researching AWS WAF, CloudWatch, SNS, and anomaly detection models to better understand how to monitor, alert, and protect systems on the cloud.

In addition, the career sharing session helped me build a clearer learning plan, focusing on system fundamentals, networking, Linux, containers, CI/CD, and cloud before going deeper into advanced technologies.

## 6. Personal reflection

The **FCJ Community Day - From Zero to Cloud Hero** provided me with many practical and applicable pieces of knowledge. Although the topics belonged to different areas, they were all related to building, deploying, operating, and protecting modern systems.

I was impressed by the way the speakers presented from practical perspectives, making the content easier to understand and not too theoretical. In particular, the topics about Docker, WebSocket, GraphRAG, and security on AWS gave me many ideas to apply to my personal projects as well as my Cloud learning process.

## 7. Conclusion

Overall, the event helped me expand my knowledge of AWS, Docker, WebSocket, GraphRAG, Machine Learning in security, and career development in the systems field. After the event, I realized that a modern system does not only need correct code, but also needs a suitable architecture, scalability, a stable deployment process, monitoring mechanisms, and effective security layers.
