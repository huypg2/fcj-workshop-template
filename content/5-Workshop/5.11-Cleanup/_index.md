To prevent unwanted charges on your AWS account after completing the workshop, follow these steps to delete all the provisioned resources in the correct order.

---

### Step 11.1: Delete ECS Services & Cluster
1. Open the **ECS** console -> Click **Clusters** -> Select **`pg-cluster`**.
2. Select the **Services** tab:
   - Check the box next to **`pg-frontend-service`** -> Click **Delete** -> Type the confirmation phrase to delete.
   - Check the box next to **`pg-backend-service`** -> Click **Delete** -> Confirm deletion.
3. Wait for the running container tasks to stop completely.
4. Click **Delete cluster** in the top right corner to clean up the cluster resources.

---

### Step 11.2: Delete ALB & Target Groups
1. Open the **EC2** service -> Scroll down to **Load Balancing** in the left menu:
   - Click **Load Balancers** -> Check the box next to **`pg-alb`** -> Click **Actions** -> Click **Delete load balancer** -> Confirm deletion.
   - Click **Target Groups** -> Select both **`tg-frontend`** and **`tg-backend`** -> Click **Actions** -> Click **Delete** -> Confirm deletion.

---

### Step 11.3: Delete RDS PostgreSQL Database
1. Open the **RDS** service -> Click **Databases** in the left menu.
2. Select the database instance **`pg-db`**.
3. Click **Actions** -> Click **Delete**.
4. Configure deletion settings:
   - **Uncheck** *"Create final snapshot?"* (Saves costs by not storing a final backup).
   - **Check** *"I acknowledge that..."*.
   - Type `delete me` in the confirmation box.
5. Click **Delete** and wait for the database deletion to finish (typically takes 5-10 minutes).
6. Click **Subnet groups** in the left menu -> Select and delete **`pg-db-subnet-group`**.

---

### Step 11.4: Delete NAT Gateway & Release Elastic IP
NAT Gateways incur hourly charges and must be deleted immediately:
1. Open the **VPC** service -> Click **NAT gateways** in the left menu.
2. Select **`pg-nat`** (or the NAT Gateway generated by the VPC wizard).
3. Click **Actions** -> Click **Delete NAT gateway** -> Confirm deletion.
4. **Release Elastic IP:**
   - Click **Elastic IPs** in the left menu of the VPC console.
   - Select the EIP that was associated with the NAT Gateway.
   - Click **Actions** -> Click **Release Elastic IP addresses** -> Click **Release** to confirm. *(Unreleased Elastic IPs incur charges if they are not associated with running resources).*

---

### Step 11.5: Delete VPC
1. Under the **VPC** service, click **Your VPCs** in the left menu.
2. Select the VPC **`pg-vpc`**.
3. Click **Actions** -> Click **Delete VPC**.
4. AWS will automatically analyze and delete the subnets, internet gateways, route tables, and security groups associated with this VPC. Click **Delete VPC** to confirm.

---

### Step 11.6: Delete ECR Repositories, S3 Bucket, and KMS Key
1. **ECR Repositories:** Open the **ECR** console -> Check `pg-combined-backend` and `pg-frontend` -> Click **Delete** to remove the registries.
2. **S3 Bucket:**
   - Open the **S3** console -> Click into your bucket **`pg-db-backups-795938553282-ap-southeast-1-an`**.
   - Click **Empty** to remove all backup files -> Type the confirmation phrase.
   - Go back to the S3 bucket list -> Select the bucket -> Click **Delete** -> Type the bucket name to confirm deletion.
3. **KMS Key:** Open the **KMS** console -> Click **Customer managed keys** -> Select **`pg-s3-export-key`** -> Click **Key actions** -> Click **Schedule key deletion** -> Set the minimum window (7 days) -> Confirm.

Congratulations on successfully completing the workshop and safely tearing down all resources!
