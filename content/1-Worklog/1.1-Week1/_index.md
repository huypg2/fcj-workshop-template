### Week 1: Cloud Kickoff, AWS Account, IAM, VPC, EC2, and Development Tools

### Week 1 Objectives:

* Understand the overview of cloud computing, cloud service models, and the AWS ecosystem.
* Get familiar with AWS account setup, AWS Management Console, AWS Budgets, and AWS Support.
* Study foundation components including IAM, VPC, EC2, EBS, Cloud9, and AWS CLI.
* Practice basic operations to prepare the learning environment, resource management, and later labs.

| Day | Task | Start Date | Completion Date | Reference Material |
| --- | --- | --- | --- | --- |

### Week 1 Results:

* Completed the initial understanding of cloud computing and why cloud platforms are widely used in modern system deployment.
* Understood the major AWS service groups such as Compute, Storage, Database, Networking, Security, and Monitoring.
* Learned how to create an AWS account, access the AWS Console, search for services, change Regions, and check active resources.
* Configured AWS Budgets to monitor cost and formed the habit of controlling resources from the beginning of AWS practice.
* Understood the role of AWS Support and AWS documentation in self-learning, troubleshooting, and checking service configuration.
* Understood the main components of IAM, including Users, Groups, Policies, and Roles, together with the least privilege principle.
* Understood how VPC creates a private network on AWS and how subnets, route tables, internet gateways, and security groups control traffic.
* Understood EC2 concepts such as AMI, instance type, key pair, and EBS volume for launching virtual servers.
* Understood why IAM Role for EC2 is safer than placing access keys directly inside applications.
* Got familiar with AWS Cloud9 and AWS CLI to operate faster in later hands-on labs.
* Built the initial foundation in account management, access control, networking, and virtual servers for later AWS services.
